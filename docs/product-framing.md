# Product Framing

## One-sentence framing

Pullstart is a repo-aware onboarding agent that helps developers go from cloned repo to working local setup through a contract-driven, verification-aware flow.

## Why this product exists

Repo onboarding is often a bad mixture of:

- stale README steps
- hidden env assumptions
- implicit toolchain versions
- service dependencies discovered only after failure
- tribal knowledge held by one teammate

Pullstart exists to turn that into a declared contract plus an agent that can inspect, plan, execute, verify, and explain.

## Sharp positioning

Pullstart is:

- onboarding-first
- contract-driven
- repo-aware
- agent-guided
- verification-aware

Pullstart is not:

- a broad platform for every developer workflow
- a workflow platform
- a memory product
- a generic code agent replacement

## Current strategic bet

If Pullstart can remove real clone-to-working friction on one credible repo type, it earns the right to generalize later.

If it cannot do that, broader platform ambition is noise.
